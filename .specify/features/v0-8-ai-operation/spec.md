## Feature Summary

把"AI 礼物经营助手"从 MVP（一次性 AI 选品 + 发布）升级为**持续经营助手**：

```
[每日]  今日经营计划 + 节日机会
   ↓
[每周]  一周经营计划 + 风格学习 + 发布时间优化
   ↓
[每次]  商品替换提醒 + 多平台内容分发
   ↓
[复盘]  AI 经营复盘 ＋ 一周完成度
```

V0.8 不再做新功能堆叠，而是把已有的 10 个 MVP 页面升级为"记住用户 / 主动推送 / 跨任务" 的统一体验。

## User Flows

### Flow 1：每日晨起（命中 P0，#1 + #2）

1. 用户打开 AI 对话首页；
2. 顶部出现"📅 今日计划：3 条任务 / 1 个节日机会"卡片；
3. 点击 → 跳转 `pages/week-plan`，展示 7 天任务 + 当日节日（如母亲节倒计时）；
4. 用户点击"开始" → 走 MVP 现有 AI 选品 → 发布链路。

### Flow 2���风格学习（命中 P1，#3）

1. 用户在 MVP 内容生成工作台选完商品；
2. AI 给出 3 版文案 + 显示"风格 chip：自然分享 / 实用推荐 / 情绪价值 / 个性化"；
3. 用户点"个性化" → AI 用 `memory-store` 的 `styleProfile` 重新生成；
4. 用户点 "📌 记住这个风格" → 写到 `memory-store.styleProfile`。

### Flow 3：发布时间优化（命中 P1，#4）

1. 发布确认页已显示封面 + 文案 + 合规状态；
2. 新增"AI 推荐发布时间"区块，给出"明早 8:30 / 今晚 21:00 / 后天中午"三档；
3. 用户选 1 档 → 写入待发布任务的 `scheduledAt`；
4. 用户点"立即发布" → 走 MVP 现有发布链路。

### Flow 4：多平台内容（命中 P2，#5）

1. 内容生成工作台文案下方出现"📤 多平台分发"标签栏；
2. 选小红书 / 朋友圈 / 视频号 → AI 自动生成对应平台的格式（如朋友圈不超过 6 行、视频号需要 hook）；
3. 用户点"复制所有"→ 一键粘贴 3 平台版本。

### Flow 5：商品替换提醒（命中 P2，#6）

1. AI 复盘页对每个历史推荐商品标"健康状态"（🟢 正常 / 🟡 同款降价 / 🔴 差评增多）；
2. 🟡/🔴 卡片显示"建议替换理由" + 1 键"换新品"按钮；
3. 用户点"换新品"→ 回到 MVP 选品流程，选新商品重新生成内容。

## Acceptance Criteria

### 宪法级（自动卡死）

- [ ] 每个 PR 必须包含 `spec.md`（本文件 + 子 spec）/ `plan.md`（人工 review 过）/ `tasks.md`（按依赖排序）
- [ ] 每个 PR 保持 `@rockcent/platform` 在 `git+https://...#platform-vX.Y.Z` 形式（不动 §6.1 §6.4）
- [ ] 金额全部用 `_fen` 整数；订单状态机不引入新状态
- [ ] AI 推荐必给理由；不出现"精准推荐 / 大数据智能 / 保证出单 / 月入过万"
- [ ] AI 记忆 / 用户偏好属于用户私有，UI 上保留一键清空入口

- [ ] 提信息用 `feat: / fix: / chore: / docs: / refactor: / test:` 前缀
